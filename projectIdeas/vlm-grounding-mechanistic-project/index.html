Where Does Visual Grounding Die in Vision-Language Models?

A research-shaped project about localizing where counting, OCR, and spatial-layout signals are preserved, lost, or ignored inside modern VLMs.

0 TL;DR

Modern VLMs can describe complex images but still fail at simple grounded tasks: counting objects, reading changed digits, understanding left/right, or following a UI instruction.

The question: where does the answer-relevant visual signal disappear?
Vision encoder, projector, early LLM layers, late language layers, or attention routing?
  • Safe version: pick two skills, build controlled clean/corrupted image pairs, and localize where the signal dies.
  • Research version: use activation patching to identify causal components, then test one lightweight intervention.
  • Critical caveat: this is less ML-systems aligned than KV-cache reuse, but much more genuinely research-shaped.

1 Core hypothesis

VLM grounding failures are probably not one phenomenon.

Working claim:
Counting may fail because visual evidence exists early but is ignored by later language layers. OCR may fail because fine-grained patch information is lost in the projector. Spatial layout may fail because 2D structure is flattened into weak 1D positional signals.

The project should be allowed to prove this wrong. Maybe all failures reduce to bad decoder attention. Maybe small VLMs fail in the vision encoder while larger VLMs fail in the language stack. Either answer is useful if supported causally.

4 Research questions

  • RQ1 - Stage localization: is the relevant signal present in the vision encoder, after the projector, in early LLM layers, or in late LLM layers?
  • RQ2 - Skill-specific failure: do counting, OCR, and spatial layout fail at the same stage or different stages?
  • RQ3 - Causal components: are there specific attention heads/layers that route visual evidence into the answer token?
  • RQ4 - Minimal intervention: can a small intervention restore the lost signal without retraining the full model?

5 Methods

Behavioral controls

Build clean/corrupted pairs where only the answer-relevant visual evidence changes. Example: same prompt, one object added; same invoice, one digit changed; same layout, button moved left/right.

Localization
  • Probe vision encoder features for the answer.
  • Probe projected visual tokens.
  • Probe residual stream across LLM layers.
  • Measure attention from answer token to visual tokens.
  • Measure visual-vs-text attention balance.
Causal tests

Use activation patching: run clean and corrupted inputs, then patch clean activations into corrupted runs. If the answer recovers, that component carried causal information.

  • Patch visual token embeddings after projector.
  • Patch early LLM residual stream.
  • Patch specific attention head outputs.
  • Patch attention patterns from answer token to image tokens.
Interventions
  • MAS-style visual attention floor: force minimum attention budget to image tokens.
  • VAR-style attention redistribution: remove sink tokens and redistribute attention.
  • Projector-side patching: preserve or reinject high-loss patch embeddings.
  • Small targeted fine-tune: train only on synthetic counting/spatial data and test transfer.

8 Scope & phases

Week 1

Reproduce + choose task

Get Spectra/hooks running. Run one counting task. Decide counting + OCR or counting + spatial.

Week 2

Controlled data

Generate 500-2000 clean/corrupted pairs with fixed prompts and known answer changes.

Weeks 3-4

Localization

Probe signal across vision encoder, projector, early LLM layers, and late layers.

Weeks 5-6

Causal patching

Patch clean activations into corrupted runs and identify layers/heads that recover answers.

Week 7

Intervention

Try MAS, VAR, or projector-side correction. Measure if the diagnosed failure improves.

Week 8

Write-up

Mechanism diagram, layer plots, patching heatmaps, intervention ablation, concise report.

9 Expected outcomes

  • Minimum viable result: counting and OCR/spatial signals degrade at different stages of the VLM stack.
  • Strong result: a specific class of heads/layers causally mediates grounded reasoning, and a cheap intervention recovers failures.
  • Very strong result: the same visual grounding/routing circuit transfers across tasks, or counting/OCR/spatial use separable circuits.

11 Related papers

Closest prior art

Counting Circuits: Mechanistic Interpretability of Visual Reasoning in LVLMs closest

arXiv 2603.18523

Introduces Visual Activation Patching and HeadLens; finds visual grounding, cross-modal routing, counting aggregation, and difficulty/existence heads. Strongest evidence that VLM visual reasoning has identifiable circuits.

Paper

Counting to Four is still a Chore for VLMs

arXiv 2604.10039

Shows count-relevant visual evidence is strongest near the projection stage and degrades in later LLM layers; proposes Modality Attention Share (MAS). Direct support for the "model sees it early, then ignores it" story.

Attention and projection failure mechanisms

See What You Are Told: Visual Attention Sink in Large Multimodal Models

ICLR 2025 / arXiv 2503.03321

Finds irrelevant visual sink tokens that absorb attention. Proposes Visual Attention Redistribution (VAR), moving attention away from sink tokens toward useful visual information.

PaperICLR poster

Lost in Embeddings: Information Loss in Vision-Language Models

Findings of EMNLP 2025

Shows the connector/projector can distort visual representation geometry; k-nearest-neighbor relationships diverge by 40-60% after projection. Suggests grounding can die before LLM reasoning begins.

ACL pagearXiv
Spatial reasoning failures

Spatial Reasoning is Not a Free Lunch: A Controlled Study on LLaVA

arXiv 2603.12545

Argues spatial failures are tied to CLIP-style encoders, 1D flattening of image patches, and weak 2D positional structure. Good architecture-level diagnosis.

Beyond Semantics: Rediscovering Spatial Awareness in VLMs

Introduces Position Sensitivity Index, Cross-Modality Balance, and RoPE Sensitivity probes. Finds visual/text norm imbalance can suppress positional signals; tests targeted interventions.

12 Open decisions

  • Which pair of skills? Recommendation: counting + OCR if you want docs/UI relevance; counting + spatial if you want purer multimodal reasoning.
  • Which model first? Recommendation: SmolVLM for debugging, Qwen2.5-VL-3B for main results.
  • Which intervention? Recommendation: MAS-style visual attention floor first, because it directly tests the language-layers-ignore-vision hypothesis.
Critical read: this is the most research-shaped option so far.
It is weaker for ML-systems job signaling, but much stronger for building actual research taste.
